# GitHub Review Checklist

Use this checklist to keep review output consistent.

- Confirm pull request scope and intent.
- Check for functional regressions and breaking changes.
- Check test coverage for changed behavior.
- Check error handling and edge-case handling.
- Check security-sensitive changes (auth, secrets, permissions, input validation).
- Check migration or compatibility risks.
- Summarize top risks first, then lower-priority concerns.
- Include specific remediation suggestions when possible.
